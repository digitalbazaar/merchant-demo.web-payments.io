Merchant Demo (HTTP API)

This site demonstrates a basic implementation of a merchant site which can be used to produce payment requests and consume payment responses. This demo is designed for non-interactive payments use cases. For example, auto payment of utility bills, business-to-business payments, a car paying for a parking spot (Internet of Things), etc. For more information, see the Web Payments HTTP API specification.

If you just want to try the Web Payments HTTP API out, the Payment Mediator Demo is probably the best place to start.

If you are a software developer and want to test your software against a Web Payments HTTP API compliant merchant, the following things may be of interest to you:

  • Attempt to access a for-pay resource. Try curl {{$root.app.config.data.baseUri}}/movies/dr-strangelove
  • Retrieve the payment request for that resource. Try curl {{$root.app.config.data.baseUri}}/movies/dr-strangelove/requests
  • Process the payment request via a payment app, which should result in a payment response.
  • Do an HTTP POST of a payment response to {{$root.app.config.data.baseUri}}/movies/dr-strangelove/payments.

The source for this demonstration can be found on Github under the digitalbazaar/merchant-demo.web-payments.io project.
